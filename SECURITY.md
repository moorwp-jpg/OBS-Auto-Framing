# Security Policy

## Supported Versions

Only the latest Preview release is supported for security fixes and security-relevant bug reports.

## Reporting A Vulnerability

Report suspected vulnerabilities through GitHub Issues and apply or request the `security` label. Do not include exploit code, secrets, private URLs, or sensitive log contents in a public issue.

Please include enough context to reproduce and triage the issue when it is safe to share:

- A short description of the impact.
- Steps to reproduce.
- OBS version.
- Windows version.
- Camera/source type.
- Relevant OBS log snippet with sensitive details removed.
- Plugin runtime status.

If the issue may expose sensitive details, open a minimal public issue first and say that private details are available on request.
