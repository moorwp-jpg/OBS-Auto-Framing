# OBS Auto Framing v0.1.0 Preview

This is a preview release of `obs-auto-framing`, a native OBS Studio video filter for automatic virtual crop and pan around people in frame. It is intended for early testing and feedback before a more polished installer-based release.

## Installation

1. Download `obs-auto-framing-v0.1.0-windows-x64.zip` and its `.sha256` checksum from this GitHub Release.
2. Close OBS Studio.
3. Extract the zip into your OBS install root, usually `C:\Program Files\obs-studio`.
4. Start OBS, select a video source, open Filters, and add the `Auto Framing` video filter.

The release zip uses the OBS folder layout directly, so installation is an extract-and-merge operation. An installer may be added later after the preview zip flow is validated.

## Defaults

- Bundled model: `YOLOX-Tiny`
- Detection Model: `Balanced - YOLOX-Tiny`
- Tracking Algorithm: `ByteTrack`
- Recommended preset: `Presenter Smooth`

The default package includes `onnxruntime.dll` and `yolox_tiny.onnx`, so normal users should not need to select a custom model path.

## Optional Models

`YOLOX-Nano` and `YOLOX-S` are optional model choices. `YOLOX-S` can be more accurate in some scenes, but it is larger and slower on CPU. If OBS feels CPU-bound or runtime status shows slow inference, switch back to YOLOX-Tiny or increase Detection Interval.

## Known Limitations

- Windows x64 only.
- CPU inference only; no GPU backend yet.
- No installer yet; the preview is distributed as a zip.
- YOLOX-S may be too slow for some CPU-only systems.
- Detection quality depends on lighting, camera angle, source resolution, occlusion, and how visible the person is.
- Custom ONNX models must be YOLOX-style models with COCO person class index `0`.
- This preview has manual OBS test coverage, but broader hardware and scene coverage is still needed.
