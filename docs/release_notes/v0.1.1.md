# OBS Auto Framing v0.1.1 Preview

This preview patch release expands ONNX capture compatibility for OBS sources that output planar, packed, or 10-bit YUV formats.

## Fixes

- Added ONNX capture support for `I422`, `I444`, `AYUV`, `I010`, and `P010` source frames.
- Converted `I010` and `P010` 10-bit samples safely to 8-bit RGB for detector input.
- Improved unsupported source frame logging with the numeric OBS format, readable format name, frame size, and linesizes.
- Updated runtime status so unsupported capture formats explain that model initialization is blocked until a supported frame is captured.

## Installation

1. Download `obs-auto-framing-v0.1.1-windows-x64.zip` and its `.sha256` checksum from this GitHub Release.
2. Close OBS Studio.
3. Extract the zip into your OBS install root, usually `C:\Program Files\obs-studio`.
4. Start OBS, select a video source, open Filters, and add the `Auto Framing` video filter.

## Troubleshooting

If runtime status reports an unsupported source frame format, check the OBS log for the readable format name. Switching the camera/source format to `NV12`, `YUY2`, `I420`, or `BGRA` is still a good fallback, and disabling HDR/10-bit output can help on older plugin versions.
